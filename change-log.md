# Change Request Log
**Project:** Cybersecurity Risk Assessment and Mitigation Strategy for an E-Commerce Platform  
**Last Updated:** May 2026  

---

## When Is a Change Request Required?

A formal change request is needed for any change that affects:
- Project **scope** (what we agreed to do)
- Project **schedule** (the finish date)
- Project **budget** (cost)
- Any **deliverable** already approved

> Small clarifications or typo fixes do NOT require a formal change request.

---

## 5-Step Change Request Process

| Step | Action | Who | Timeframe |
|------|--------|-----|-----------|
| 1 | Identify & submit the Change Request Form | Anyone (team member, sponsor, stakeholder) | Immediately when change is identified |
| 2 | Log & assess impact on scope, schedule, cost, quality | Project Manager | Within 2 working days |
| 3 | Review by Change Control Board (CCB) | Project Manager, Lead Security Analyst, Platform Owner | Weekly meeting or via email |
| 4 | Implement approved change; update plan, Gantt, risk register | Project Manager + assigned team member | As per agreed timeline |
| 5 | Track in Jira; mark "Closed" when done; add lessons learned | Project Manager | Upon completion |

## Change Request Form Template

| Field | Details |
|-------|---------|
| Change Request ID | e.g., CR-001 |
| Date Submitted | dd/mm/yyyy |
| Requested By | Name and role |
| Change Description | What, why, and expected benefit |
| Affected Area | Scope / Schedule / Budget / Deliverable |
| Urgency | Low / Medium / High |
| Impact on Schedule | +X days / no change |
| Impact on Cost | +$X / no change |
| Impact on Scope | Adds / removes / modifies which item? |
| CCB Decision | Approved / Rejected / Deferred |
| Decision Date | dd/mm/yyyy |
| Approval Signature | CCB chair or project sponsor |
| Implementation Status | Not started / In progress / Completed |
| Closure Date | dd/mm/yyyy |
